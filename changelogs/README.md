# Changelogs

Per-batch changelog files generated by the `/changelog` command.

## How it works

Run `/changelog [input]` where input is either a file path with raw notes or a free-text dump. The command invokes the `changelog-refiner` agent (see `.claude/agents/changelog-refiner.md`) which:

1. Parses your raw notes into atomic change items
2. Categorizes them using Keep-a-Changelog conventions (Added / Changed / Fixed / Removed / Deprecated / Security)
3. Polishes each one-liner
4. Scans `CLAUDE.md`, `.claude/commands/`, `.claude/agents/`, `context/`, `data_sources/`, and `README.md` to flag documentation that may need updates
5. Writes a dated file here (`YYYY-MM-DD-[slug].md`)
6. Appends a Keep-a-Changelog summary to the root `CHANGELOG.md`

## File format

Each file contains:

- **Summary** — one-paragraph human-readable recap
- **Changes** — categorized, polished one-liners with optional detail and affected paths
- **Documentation Impact** — table of files/sections that likely need updates, with concrete suggestions
- **Breaking Changes** — explicit migration notes when applicable
- **Raw Notes** — verbatim original input, collapsed in a `<details>` block for traceability

## What this system does NOT do

- Touch git in any way (no commits, no diffs, no log reads)
- Modify documentation automatically — it only flags what likely needs updating
- Publish or push anything anywhere

Doc updates and commits are always your call.
